Categorical grid — one mark per cell

A band × band point plot (3 columns × 3 rows). Both positional channels are categorical, so a mark lives in a discrete cell.

  • Drag a circle to move it to another cell (2D drag over two band scales — it snaps to the nearest cell).
  • Click an empty cell to create a circle there.
  • Alt-click a circle to delete it.

Both axes are categorical on a circle mark, so they resolve to d3.scalePoint automatically (a dot sits on the tick) — not scaleBand, which is for bars.

The rule is unique({ field: ["x", "y"] }) — a composite key over both axes, so at most one mark per (x, y) cell. Two marks may share a column or a row, just not the same cell. A create on an occupied cell, or a drag onto one, is rejected (the mark stays put). This is a data-layer invariant, so it holds for both create and move.
