# Point

<Lead>`point` is the channel-driven scatter mark: every channel — x, y, size, fill, stroke, angle — resolves through the global scales. Default shape is a circle; `shape: 'square'` emits a centred rect (side `2 × size`) that can rotate about its centre.</Lead>

<Section id="heatmap" title="Heatmap of circles / squares">

Put a category on **both** x and y and each dot centres at that intersection — a grid heatmap. `fill` encodes the value as colour and `size` as radius, so magnitude reads twice; `shape: 'square'` swaps circles for centred squares. Reach for `rect` instead when you want filled cells that tile the plane. The `size` scale is invertible, so `resize()` on it drags a dot's radius back to a value — its colour re-derives from the same field.

<Section id="creating" title="Adding points">

`point` is the canonical creatable mark: a plane gesture inverts the pointer through the x and y scales to mint a datum, exactly the inverse of how those channels encode it. Because a move-drag lives on the same mark here, the creator uses `gesture: "dblclick"` so it doesn't race the drag. See [Create · Remove · Anchors](/editing/existence) for the full model and the can/can't-create matrix.

<Section id="angle" title="Rotating squares and ticks">

An `angle` channel orients a mark about its centre. Degrees are math convention (0° = +x, counterclockwise); `rotate()` inverts the pointer through the same scale. Squares use `point({ shape: 'square' })`; short `tickX` / `tickY` segments (with `length`) rotate the same way — useful for directional markers on a scatter.
