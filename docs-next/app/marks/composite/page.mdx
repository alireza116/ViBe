# Composite

<Lead>A **composite** mark is a glyph: a named group of ordinary marks over the chart's one dataset. Group-level `channels` (and style / `angle` shorthands) trickle into every part at desugar time — declare a shared orientation or position once; a part's own channel for the same name wins; inherited `edit`s attach to the **last** part only (visuals first, handles last). Each part still has its own channels and shorthands: arm geometry, per-part stroke, a hub's fill. Composite is a desugaring, not a new kind of mark — it returns its parts as plain marks, and `Elicit` flattens them. Because each handle is its own mark, dragging one cannot move another: a gesture routes only to the mark you touched.</Lead>

<Section id="lollipop" title="Lollipop — drag the tip">

A stem (a span rule from the baseline to the value) plus a draggable tip. Two marks: the `ruleX` draws the stem and carries no edit, so it is inert; the `point` is the handle and edits `value`. Drag a tip and its stem re-derives, because both marks read the same row.

<Section id="error-bar" title="Error bar — move the dot, resize the interval">

Three editable fields on one row: `mean` (the dot), `lo` and `hi` (the caps). Each is a separate mark, so each edits a plain `y` channel and dragging a cap moves only that end — no handle arbitration to write. The whisker is a `ruleX` spanning lo..hi; it follows the caps because it reads the same rows.

<Section id="plus" title="Group channels + per-part channels">

Two layers of binding. **Group** `channels` (here `x`, `y`, `angle`) trickle into every part so the glyph shares one position and one orientation. **Each part** still declares its own channels and shorthands — arm length, stroke colour, a hub’s `fill` / `size`. A part key wins on conflict. Inherited `edit`s attach to the last part only, so one `rotate()` spins the whole glyph without double-applying.

<Section id="creating" title="Creating a whole glyph at once">

Creation is [mark-agnostic](/editing/existence#what-can-create): a plane gesture mints one row, and every part of the glyph encodes it — so one create makes a whole composite appear. The rule that a whole-dataset edit belongs on **exactly one mark** is what makes this clean: put `create` on a **single part** (here the tip), and a click appends one row, not one per part. The inert stem re-derives from the same row, so the stem and tip show up together.
