# Overview

<Lead>VibeJS builds charts people answer with — drag a bar, place a point — and you get structured data back. A chart is a declarative `Elicit(spec)`: one **schema**, one **dataset**, and a list of **marks**. Every example on these pages runs the exact code beside it. This page is the wiring tour; [Core concepts](/concepts) walks the mental model.</Lead>

<Section id="anatomy" title="A minimal chart">

Three pieces make a static chart: `schema` (what each field is and its domain), `data` (the rows), and `features` (marks that encode those rows). That is the whole model until you attach an edit.

<Section id="encode-edit" title="Encoding, and its inverse">

Attach an edit to a channel and a gesture writes that channel back through the **same** scale. The bar below is identical to the one above, except `y` now carries `edit: drag()` — so encoding (data → height) gains its inverse (drag → data).

<Section id="reading" title="Reading the answer">

Committed data leaves the chart through `onChange`, `getData()`, or `chart.on("change", …)`. Hover previews never fire these — only a finished edit does.
