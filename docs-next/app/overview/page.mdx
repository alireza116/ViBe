# Overview

<Lead>A chart is a declarative `Elicit(spec)`: a list of **marks**, each with **channels** that map data fields to visuals. Every example on these pages shows the **exact code that drew the chart beside it** — the snippet is run verbatim. This page is the 30-second API tour; the sidebar goes deep on each mark and feature.</Lead>

<Section id="anatomy" title="A minimal chart">

An Elicit spec is a renderer plus a list of features (marks). A mark binds channels (x, y, fill, size, …) to data fields through scales. That is the whole model: encode (data → visual) one way, edit (gesture → data) the other.

<Section id="encode-edit" title="Encoding, and its inverse">

Attach an edit to a channel and a gesture writes that channel back to the data through the same scale. The bar below is identical to the one above, except y now carries edit: drag() — so encoding (data → bar height) gains its inverse (drag → data).
