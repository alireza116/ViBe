# Guides

<Lead>A guide is a graphical mark that isn’t bound to a row the way a plot mark is: it draws an annotation. But it is still allowed to **depend on the data** — every guide option may be a literal _or_ a function of the guide context (`{ data, scales, features, ui, width, height, stage }`), so a mean line is one line of spec. Guides rebuild every render and never capture a gesture. `guides.rule` draws a reference line at a data value, `guides.region` shades a band between two values, and `guides.proximity` visualizes what a nearest-pick edit has selected. (An edit’s own constraint bounds draw automatically via `guide: true`.)</Lead>

<Section id="annotations" title="Rule & region">

A shaded target band (region) and a reference line (rule) annotate a chart. They sit under the marks and follow the same scales, so they stay put as points move.

<Section id="live" title="Guides derived from the data">

A guide option can be a function of the guide context, so an annotation can be computed from the rows it annotates. Here one rule is a fixed target and the other is the running **mean** — recomputed on every commit, because guides rebuild each render. The band tracks the spread the same way.

<Example
  code={aMeanLineThatChasesTheBars}
  title="A mean line that chases the bars"
  blurb={"guides.rule({ y: ({ data }) => d3.mean(data, (d) => d.y) }) — the annotation reads the dataset."}
  try={(
    <>
      <b>Drag</b> a bar — the fixed target holds, the mean line and the band follow.
    </>
  )}
/>

</Section>
