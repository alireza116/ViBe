# Widgets

<Lead>Opinionated **survey instruments** — option rings, cell grids, tracks — each a pure recipe over the core API. They add no interaction surface: the look lives entirely in the [guide](/guides) layer, and the behaviour is a normal mark + a normal edit on the [probe](/editing/probe) driver + a normal constraint. The Likert scale below is shown both as the one-line widget **and** as its plain-API twin, so you can see exactly what a widget expands to — every other instrument here is just the widget. Each factory returns an **ElicitSpec**: render with `Elicit(widgets.likert({…}))`.</Lead>

<ApiReference entries={api} />

<Section id="likert" title="Likert scale — widget & plain API">

A point on a band of options + create (probe) + `count(\{ max: 1, replace \})`. Hover to preview the answer, click to fill it. The plain-API twin draws the same look with inline `guides.custom` helpers and the same mark + edit + constraint — the widget is only a shorthand for it.

<Section id="choice" title="Multiple choice">

<Section id="matrix" title="Question matrix">

Band y of questions × band x of options. `toggle()` on the (x, y) tuple names a cell; `unique(\{ field: "question", replace \})` keeps one answer per row.

<Section id="styles" title="Customizing styles">

For a whole styled family, pass a **`theme`** — every widget accepts one, and it restyles the answer ink, chrome, and affordances together (see [Theming](/theming)). For one-off tweaks, the look is just guides + mark paint, so you can repaint the answer mark or swap the affordance in the [plain-API twin](#likert) above — behaviour (edits / constraints) stays put.

<Example
  code={themedWidget}
  title="Restyle with a theme"
  blurb="widgets.slider({ …, theme: themes.survey }) — one object restyles the instrument."
  try={"drag the knob — only the look changed."}
/>

</Section>
