# Responsive sizing

<Lead>By default a chart draws at the exact `width` / `height` you give it, in pixels. The `responsive` option makes it fit its parent instead. `"scale"` wraps the SVG in a **viewBox**, so the browser stretches one drawing to the container and the aspect ratio holds. `"reflow"` measures the parent and **redraws at native pixels** on resize, so text stays crisp, the width tracks the container, and the height stays the value you set. Either way, the `width` / `height` you pass is the design size.</Lead>

<ApiReference entries={api} />

<Section id="scale" title="Scale (viewBox)">

`responsive: "scale"` keeps the chart's 480×300 aspect ratio and stretches it to fill the column; text and strokes scale with it. Reach for it when a chart should grow and shrink with its container and some drift in aspect is fine.

<Section id="reflow" title="Reflow (ResizeObserver)">

`responsive: "reflow"` measures this card and redraws at native pixels: the width tracks the container while the height stays 300. Resize the window and the bars re-lay out crisply, with no scaling blur. Call `el.destroy()` on unmount.

<Section id="fixed" title="Fixed (default)">

With no `responsive` option (or `responsive: "fixed"`) the chart is exactly 320×240 pixels, whatever the container. Use it when a chart must be a precise size.
