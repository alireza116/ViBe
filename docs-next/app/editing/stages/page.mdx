# Stages

<Lead>Multi-step elicitation: "first do X, then do Y". An edit may carry a numeric `stage`; it is active only when it equals the chart's current stage (edits with no stage are always active). This is a uniform gate applied to every edit — the same shape as gesture matching, not a new interaction mode — so stages compose with every mark and edit. Advance from the container: `nextStage()`, `setStage(n)`, `getStage()`, `on("stage", cb)`.</Lead>
