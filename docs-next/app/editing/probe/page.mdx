# Probe

<Lead>The **probe / settle** flow: the pointer **probes** a value and the proposal follows the cursor as an inert **ghost** — the committed mark stays put, so nothing flickers — while `getData()` still reports the last committed belief. A **commit** settles it, and there are two ways to commit, so both natural gestures work: **move-then-click** (no button held), and **grab-and-drag** — press on the mark, drag, release. Give any edit `pick: "probe"` and it works this way, so the same primitive drives a correlation line, a slider knob, a Likert answer and a dot plot's tentative token. Add a `stage` and each commit settles that stage's field and advances to the next — "set the line, now open the cone".</Lead>
