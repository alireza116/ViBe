# Editing — the inverse of encoding

<Lead>Attach an edit to a channel and a gesture writes that channel back to the data through the same scale. An edit is a small descriptor — `gesture` (drag / click / dblclick), `pick` (direct / nearest / plane), `when` (arbitration), `constrain`, and `guide` — with an `apply` that does the inverting. The named factories (`move`, `resize`, `rotate`, `cycle`, …) are just presets over `makeEdit` with a fixed `apply`; see [Gestures](/editing/gestures) for the full set. Place an edit on a channel (`channels.y.edit`) or at mark level (`edits: [...]`).</Lead>

<ApiReference entries={api} />

<Section id="move" title="Move a value">

The canonical edit: `move()` inverts the pointer on each positional channel it governs. On a bar's `y`, a drag rewrites the value.

<Section id="pick" title="Pick — how an edit finds its target">

`pick: "direct"` uses the mark under the pointer; `pick: "nearest"` grabs the closest mark within a pixel threshold, so small marks are reachable from nearby empty space. `guide: true` self-draws the snap ring.
