# Create · Remove · Anchors

<Lead>Creating and deleting data are edits like any other. `create` is a plane gesture that inverts the pointer through the positional channels to mint a datum; `remove` deletes the target. For connected paths, `edit.line.anchor` adds one point to the nearest line, `edit.line.newSeries` seeds a whole line, `edit.line.draw` lays a line down as you drag, and `edit.line.removeSeries` deletes a whole line. Multiple edits on one gesture are arbitrated by `when`. **Declare a whole-dataset edit on exactly one mark.** A plane gesture carries no node, so it fans out to _every_ mark’s plane-pick edits — put `create` on two marks and one click appends two rows. (Direct-pick edits like `drag` are immune: they route to the mark you actually touched.) The engine warns in dev builds if you do this.</Lead>

<Section id="existence" title="Create, remove & move together">

Mark-level create (dblclick) + remove (Alt-click) + a move drag, all on one mark. When two edits share a gesture, when decides which claims the event.

<Section id="anchors" title="Anchors — building a connected path">

On a line, edit.line.anchor() adds a point to the nearest series (or starts a new one from empty space), and edit.line.newSeries() seeds a whole line from sampled positions. Order is tracked so the path stays reproducible.

<Example
  code={clickToAddAnAnchor}
  title="Click to add an anchor"
  blurb={"edit.line.anchor({ into: \"nearest\" }) extends the connected path in click order — near or far, it stays one line (into: \"new\" starts a fresh one)."}
  try={(
    <>
      <b>Drag</b> a point, or <b>click</b> empty space to add the next anchor.
    </>
  )}
/>
