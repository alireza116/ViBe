# Create · Remove · Anchors

<Lead>Create and remove work like any edit: `create` inverts the clicked pixel into a datum through the same scales `encode` uses, then appends it to the dataset — every mark reading those rows encodes it, composite glyphs included. `edit.line.anchor` / `newSeries` / `draw` extend that to connected paths; `edit.geo.create` / `createRect` / `draw` on [Geo](/marks/geo) invert a map projection instead. `remove` and `edit.line.removeSeries` take rows back.</Lead>

<ApiReference entries={api} />

<Section id="what-can-create" title="What can — and can't — be created">

Creation writes dataset **rows**, so it belongs on a **data mark positioned by an invertible axis** — which is nearly every mark. Only **guides** (they carry no per-row data) and **derived** marks (their position is computed from the whole dataset, not one field) are excluded. If you put a creator where it can't place a datum, the dev build warns you.

<Section id="gesture-rule" title="Picking the gesture">

`create` defaults to `gesture: "click"`. Give it `gesture: "dblclick"` when a move-drag lives on the same mark, so a press-and-move reshapes the point you're on instead of racing the create — every "create + move" example below uses `dblclick` for this reason. A plain `click` works when nothing else claims the plane.

</Section>

<Section id="create" title="Create — click to add">

`create` is its own edit: a plane click inverts the pointer through the positional channels and appends a datum. Constraints still apply — `count(max)` caps the total.

Declare a creator on exactly one mark. A plane gesture carries no node, so it fans out to every mark's plane-pick edits — put `create` on two marks and one click appends two rows. Direct-pick edits like `drag` are immune: they route to the mark you actually touched. The dev build warns if you don't. When multiple edits share one gesture, `when` arbitrates which one claims it.

<Section id="anchors" title="Anchors — building a connected path">

On a line, `edit.line.anchor()` adds a point to the nearest series, or starts a new one from empty space. `edit.line.newSeries()` seeds a whole line from sampled positions. Order is tracked so the path stays reproducible.
